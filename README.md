- **معرّف التطبيق:** `com.deciphertahiro.app`
- **الإصدار:** 5.0.0 (`versionCode 1`)
- **المنصة:** Android (minSdk 23 / targetSdk 35)

```bash

  -alias decipher -keyalg RSA -keysize 2048 -validity 10000
```

احفظ كلمة المرور والـ alias في مكان آمن — **ستحتاجهما دائماً للتحديثات**.

  - ⚠️ **ملاحظة أمان:** ملف `google-services.json` يحتوي إعدادات مشروعك. إن كان المستودع **عاماً (public)**، سيكون الملف مرئياً. هذا آمن عادةً للتطبيقات (المفاتيح محميّة بقواعد أمان Firebase وليست سرية)، لكن يُفضّل إبقاء المستودع **خاصاً (private)** أو ضبط قواعد أمان صارمة في Firebase Console.

- **RTL:** التطبيق يدعم الكتابة من اليمين لليسار بالكامل.

## 🐉 محرّك Hydra الهجين للعب العالمي

أُضيف محرّك **Hydra** — توليفة قوية تجمع عدة مزوّدين للعب عبر الإنترنت مع لاعبين من جميع أنحاء العالم. الفلسفة: **Firebase يبقى مصدر الحقيقة** (الحالة الموثوقة)، والباقي طبقة فوقه للسرعة والحضور والـ failover.

| المزوّد | الدور |
|---------|-------|
| **Firebase** | مصدر الحقيقة: الغرف، الحالة، النتائج |
| **Supabase Realtime** | الحضور العالمي (كم لاعب متصل الآن) + قائمة الغرف الحية |
| **Ably** | بثّ الحركات اللحظية (<50ms) |
| **PubNub** | بثّ متزامن (يعمل مع Ably معاً) |
| **Photon** | غرف لعب احترافية (يُحمّل عند توفّر الـ SDK) |
| **PeerJS** | اتصال P2P مباشر بين اللاعبين (يقلّل زمن الوصول) |

كل المزوّدين خلف واجهة واحدة `HydraNet`. تُهيّأ كلها بالتوازي، وسقوط أحدها لا يُسقط البقية (`Promise.allSettled`).

### كيف يعمل
- عند الدخول: `HydraNet.init()` يشغّل كل المزوّدين + ينضم للحضور العالمي.
- عند بدء تحدي: `HydraNet.joinRoom(id)` ينضم لكل قنوات البثّ النشطة (Ably + PubNub + Photon) معاً فوق Firebase.
- عند كل محاولة: `HydraNet.sendMove()` يبثّ الحركة عبر **كل القنوات في آنٍ واحد** (multi-cast) + P2P، مع إزالة التكرار عند الاستقبال — موثوقية ووصول أقصى.
- شارة "🌍 متصلون" تعرض عدد اللاعبين المتصلين عالمياً.

### 🔐 توصية أمان — مفتاح Ably

كل المزوّدين الستة **مفعّلون ويعملون** الآن، ومفتاح Ably موضوع مباشرة في `hydra-engine.js` كما طلبت.

⚠️ **توصية (اختيارية لكن مهمة):** المفتاح الحالي صلاحياته admin. للأمان طويل المدى يُفضّل استبداله بمفتاح **publish/subscribe فقط**:

1. [Ably Dashboard](https://ably.com/accounts) ← تطبيقك ← **API Keys** ← **Create a new API key**
2. فعّل فقط: **Publish** + **Subscribe** + **Presence**
3. ضعه في `hydra-engine.js` مكان المفتاح الحالي، ثم ألغِ القديم.

> مفتاح pub/sub آمن في كود التطبيق (هذا ما تفعله الألعاب الاحترافية)، أما مفتاح admin فيمكن لأي شخص يفك الـ APK استغلاله.

### 🔒 أمان Supabase

مفتاح `anon` العام آمن **بشرط** تفعيل **Row Level Security (RLS)** على كل الجداول في Supabase Console. بدون RLS أي شخص يستطيع قراءة/كتابة بياناتك.

### إيقاف مزوّد
في `hydra-engine.js` غيّر `enabled: false` لأي مزوّد لا تريده.

## 🔒 الموافقة على الخصوصية (UMP / GDPR)

أُضيفت **بنية موافقة جاهزة** عبر Google User Messaging Platform (UMP SDK) للامتثال لـ GDPR (أوروبا) وقوانين الولايات الأمريكية.

### ⚠️ مهم جداً — اقرأ هذا أولاً

**التطبيق لا يحتوي إعلانات حالياً**، وبنية UMP المُضافة **صامتة تماماً** حتى تُكملها أنت:

- لن تظهر أي نافذة موافقة للمستخدم إلا بعد أن تُنشئ **رسالة GDPR** في AdMob Console وتربط التطبيق بـ AdMob.
- قبل ذلك: `MainActivity` يفحص الموافقة عند التشغيل، لكن لا شيء يُعرض (لأنه لا توجد رسالة معدّة) — وهذا آمن ولا يزعج المستخدم.

### كيف تُفعّلها لاحقاً (عند إضافة إعلانات)

1. أنشئ حساب [AdMob](https://admob.google.com) واربط التطبيق `com.deciphertahiro.app`.
2. من **Privacy & messaging → GDPR** اضغط **Create message** وانشرها.
3. (اختياري) من **Privacy & messaging → US states** اضغط **Create** — تعمل تلقائياً دون أي كود.
4. أعد بناء التطبيق. الآن:
   - مستخدمو **أوروبا** يرون نافذة الموافقة تلقائياً عند التشغيل.
   - مستخدمو **المغرب وباقي الدول** غير الخاضعة: تُتجاهل النافذة تلقائياً.
   - زر **"إدارة خيارات الخصوصية"** يظهر في شريط اللغة لمن يلزمه فقط (يعيد فتح النافذة).

### ما يحدث في الكود

| الملف | الدور |
|------|-------|
| `app/build.gradle` | يضيف مكتبة `user-messaging-platform:3.2.0` |
| `MainActivity.java` | يفحص الموافقة عند التشغيل + يعرض النافذة إن لزم + جسر لزر الخصوصية |
| `index.html` | زر "إدارة خيارات الخصوصية" (يظهر تلقائياً لمستخدمي أوروبا فقط) |

### عند إضافة إعلانات مستقبلاً

في `MainActivity.java` يوجد تعليق `(مستقبلاً) تُهيّأ إعلانات AdMob هنا` — هذا المكان الصحيح لتشغيل الإعلانات **بعد** قرار المستخدم. هذا الترتيب إلزامي للامتثال: لا إعلانات قبل الموافقة.

> 🛡️ **ملاحظة:** هذا الكود لا يجمع أي بيانات ولا يعرض أي إعلان حالياً. هو فقط بنية تحتية جاهزة للمستقبل، ولا يؤثر على تجربة المستخدم الآن.

4. **البيني بعد النتائج فقط** وبحدّ تكرار (كل 3 تحديات).
5. المتجر النقدي (PayPal) يبقى مجمّداً — المكافآت مجانية عبر الإعلانات فقط (متوافق مع Google Play).

### كيف تعمل (التدفّق)

```
المستخدم يضغط "اكسب مكافأة" → watchRewardAd(key)
   → window.AndroidAds.showRewarded(key)   [JS → Java]
   → MainActivity يحمّل النوع الصحيح ويعرضه
   → عند المشاهدة الكاملة: onUserEarnedReward
   → window.onRewardEarned(key)            [Java → JS]
   → تُضاف المكافأة لحالة اللعبة + حفظ
```

1. **الشاشة الرئيسية** — بانر أخضر نابض تحت بانر صراع الأدمغة مباشرة
2. **عند نفاد الذهب** — زر "شاهد إعلاناً واكسب ذهباً" في نافذة الرصيد المنخفض
3. **عند نفاد القلوب** — زر "شاهد إعلاناً واكسب قلباً" أعلى نافذة القلوب

كلها تفتح شاشة المكافآت نفسها (`openRewardScreen`) وتعمل بكل اللغات السبع.

|------|--------|--------|
| App Open | `…4776542896` | ✅ مدمج — يظهر عند **العودة** للتطبيق فقط (ليس أول تشغيل)، بفجوة 4 ساعات، وممنوع أثناء التحدي |
| Banner | `…8202184955` | ✅ مدمج وذكي — يظهر **في القوائم فقط** ويختفي **تلقائياً أثناء اللعب** |
| Native | `…8691737172` | ❌ **لم يُدمج** — يتطلب تخطيطاً مخصّصاً معقّداً داخل WebView، لا يستحق المخاطرة. المعرّف محفوظ كتعليق للمستقبل |

**ضمانات App Open الصارمة:**
- لا يظهر عند أول تشغيل (cold start)
- لا يظهر أثناء تحدٍّ حيّ (يُبلّغ كود الويب الأصلي عبر `setInChallenge`)
- فجوة 4 ساعات بين كل ظهور
- يُحمّل التالي تلقائياً بعد كل عرض

**Banner (ذكي):** يظهر في الشاشة الرئيسية/القوائم فقط، ويختفي تلقائياً عند فتح نافذة لغز التحدي أو أثناء أي تحدٍّ نشط، عبر `updateBannerVisibility()` التي تُستدعى عند كل انتقال. يُضاف هامش سفلي تلقائياً (64px) لئلا يغطّي البانر المحتوى.

**الإجمالي: 8 وحدات مدمجة وعاملة** (5 مكافآت + بيني + App Open + Banner ذكي)، والإعلان المدمج (Native) متروك بقصد.



### ⚠️ قبل النشر على Google Play

1. في **App content → Ads** صرّح أن التطبيق يحتوي إعلانات.
2. اضبط **تصنيف المحتوى** (Content rating) ليشمل وجود إعلانات.
3. تأكد أن **سياسة الخصوصية** تذكر استخدام AdMob وجمع البيانات للإعلانات.
4. أنشئ **رسالة GDPR** في AdMob Console (وإلا لن تُهيّأ الإعلانات لمستخدمي أوروبا).
